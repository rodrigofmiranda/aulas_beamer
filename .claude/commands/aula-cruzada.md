# /aula-cruzada
## Uso: `/aula-cruzada N M`
## Exemplo: `/aula-cruzada 6 7`

---

Este arquivo é um **wrapper fino** para evitar divergência entre o fluxo do Claude e o fluxo canônico do projeto.

## FONTE CANÔNICA DESTA ETAPA

Leia e siga integralmente:

- `CLAUDE.md`
- `aula-cruzada.md`
- `memoria_projeto.md` (quando existir)

## INSTRUÇÕES

1. Execute o fluxo completo definido em `aula-cruzada.md` para as Aulas **{{N}}** e **{{M}}**.
2. Verifique consistência, continuidade, separação teoria/lab e coerência de glossários conforme o fluxo canônico.
3. Se houver qualquer conflito entre este wrapper e os arquivos canônicos, **vale o arquivo canônico**.
4. Entregue relatório consolidado com as tags de auditoria cruzada.

## REGRA DE MANUTENÇÃO

Novas regras desta etapa devem ser adicionadas em `aula-cruzada.md`, não aqui.
